# Task B5.4 — Admin endpoints (SettingsRepo, admin use cases, httpapi handlers)

You are a coding agent on **Cloud Lab Gateway**. Read `CLAUDE.md` and
`docs/ARCHITECTURE.md` first.

## Goal

Replace the `501` stubs for the admin surface with real use cases + handlers:
`GET /admin/projects`, `POST /admin/projects/{id}/quarantine`,
`POST /admin/projects/{id}/release`, `GET /admin/quota`,
`GET /admin/settings`, `PUT /admin/settings`, `GET /admin/audit`.

## 1 — SettingsRepo (pgxrepo)

`SettingsRepo` has no Postgres implementation yet. Add `pgxrepo/settings.go`
(+ `queries/settings.sql`, then `make gen-sqlc`), implementing `Resolve`,
`List`, `Put`. Mirror the existing repos. Note the `settings` PK is
`(key, scope, scope_id)` and `scope_id = ''` means global.

## 2 — admin use cases (`internal/app/admin`)

Each takes the actor and enforces role (admin-only unless noted):

| Use case | Auth | Behaviour |
|---|---|---|
| `ListProjects` | admin | `PoolRepo.ListByDomain` with optional `ki_domain_id` / `state` |
| `QuarantineProject` | admin | load project, `project.Quarantine(reason, now)`, save |
| `ReleaseProject` | admin | load project, return it to `free`, save |
| `GetQuota` | admin | `QuotaCache.Read` → `QuotaSnapshot` (compute `utilization_pct`) |
| `ListSettings` | teacher or admin | `SettingsRepo.List` |
| `PutSetting` | teacher or admin | `SettingsRepo.Put`; `global` scope ⇒ `scope_id = ''` |
| `QueryAudit` | teacher or admin | `AuditRepo.Query` from the `AuditFilter` query params |

Authorization failure → `shared.ErrForbidden`. Project saves go through
`UoW.WithTx` so domain events flush to the outbox.
