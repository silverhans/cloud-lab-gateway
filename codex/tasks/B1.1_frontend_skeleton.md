# Task B1.1 — Frontend skeleton (React + Vite + TS + Tailwind + shadcn/ui)

You are a coding agent working on **Cloud Lab Gateway**. Read `CLAUDE.md` and `docs/ARCHITECTURE.md` first. Your scope is `web/` — backend is a different stream, don't touch it.

## Goal

Stand up a React 18 + Vite + TypeScript + Tailwind + shadcn/ui frontend with three role-scoped routes (`/student`, `/teacher`, `/admin`), placeholder pages, a TanStack Query client, an SSE event subscriber utility, and a typed API client stubbed from `api/openapi.yaml`. The skeleton must build into a production bundle and serve through the existing nginx-frontend container.

Important repository nuance: the generated REST client comes from
`api/openapi.yaml`, but the live events stream is not part of that client.
Consume the nginx-routed SSE endpoint `/sse/labs` directly from the browser.

## Concrete behaviour required

### Routing + role redirect

`/` (index.tsx) calls `useCurrentUser()`:

- Loading → spinner
- Anonymous → navigate to `/login`
- `role === student` → `/student`
- `role === teacher` → `/teacher`
- `role === admin` → `/admin`

Teachers must also have access to `/admin/audit` (audit log is teacher-visible per policy). Wire role-guard at the route level, not just hidden links.

### Login page

Simple form (email, password) calling `POST /api/v1/auth/login`. On success → invalidate user query → router auto-navigates. Show validation errors from RFC 7807 `application/problem+json` responses.

### Student page (placeholder)

Show the current user's display name + role badge + the message "Лаба ещё не запущена — переходите сюда из Moodle". Add a stub button "Demo: запустить" that calls `POST /api/v1/labs` with hard-coded `course_id` / `lab_template_id` (env-driven, see below) — useful for end-to-end testing without Moodle running.

### Teacher page (placeholder)

Empty card "Лабы курса" with `Table` skeleton (columns: Student, State, Created, Cleanup at, Actions). Add an empty card "Настройки таймаутов" linking to `/teacher/settings` (page not implemented yet — render a 404 route is fine).

### Admin page (placeholder)

Three card placeholders: "Pool status", "Cluster utilisation", "Audit log". No data wiring yet — these come in B5.x.

### SSE subscriber

`useLabsStream` opens `EventSource('/sse/labs')`. Decode events with Zod
(define an `SSEEventSchema` discriminated union of `lab.state_changed`,
`lab.deploy_progress`, `check.state_changed`, `quota.snapshot`). On each
event, `queryClient.setQueryData(['labs', id], updater)`. Reconnect is
automatic (browser EventSource semantics); on close, log to console.

Do not "fix" the SSE path by editing backend or OpenAPI files in this task.
Frontend should consume `/sse/labs` as routed by nginx.

### API client

```ts
// lib/api.ts
import createClient from "openapi-fetch";
import type { paths } from "./api.gen";  // generated file — see below

export const api = createClient<paths>({
  baseUrl: "/api/v1",
  credentials: "include",  // session cookie
});
```

Generate `api.gen.ts` from `../api/openapi.yaml`:

```bash
npx openapi-typescript ../api/openapi.yaml --output src/lib/api.gen.ts
```

Add this command as `pnpm run gen:api` (or npm equivalent). Don't commit the generated file pristine — gitignore it and document the command in README.

Use shadcn defaults but wire CSS variables so we can rebrand later. ```

## Acceptance criteria

1. `pnpm install && pnpm run build` produces `web/dist/` without errors or warnings.
2. `pnpm run typecheck` is clean. **No `any` outside generated files.**
3. `pnpm run lint` is clean.
4. Navigating to `/` while anonymous → redirected to `/login`. Logging in (against a running gateway) → role-appropriate page.
5. The shadcn components in `components/ui/` are copied via `npx shadcn@latest add` and committed (per shadcn convention — they are project source, not dependencies).
6. Building the `frontend` Docker image (`deployments/Dockerfile.frontend`) succeeds and serves the SPA at `http://localhost:8080/` through nginx-proxy.
7. The bundle gzip-size for the initial route is **< 250 KB** (check with `pnpm run build`). If it's bigger, lazy-load route components.
8. README in `web/README.md` explains commands above.

## Constraints

- TypeScript `"strict": true`. No `// @ts-ignore`. No `any`.
- All form inputs labelled (a11y). All buttons have accessible names.
- Loading and error states for every async boundary (use Suspense + ErrorBoundary).
- No CSS files outside `globals.css` and shadcn's generated ones. Tailwind only.
- Use date-fns (not moment.js) for any date formatting; ship Russian locale.

## Do NOT touch

- Anything outside `web/` (except adding `web/README.md` link to root `README.md`'s "Frontend" section).
- `api/openapi.yaml` — that's the contract, not the client's concern.
