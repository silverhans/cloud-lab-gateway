## Critical implementation details

### LabRepo

- **Create / Save must flush domain events.** After the row write, call `l.PullEvents()` and for each event INSERT into `outbox` (topic = event kind) AND `audit_events`, in the **same tx**. This is exactly what `pool.go`'s `appendProjectEvent` does — mirror it. The lab events are in `internal/domain/lab/events.go`: `EventCreated`, `EventStateChanged`, `EventProjectAssigned`, `EventCleanupExtended`.
- **`Create` and the "one active lab" rule.** The schema has a partial unique index `lab_instances_one_active_per_student_course`. If `Create` hits a unique violation (pgconn error code `23505`), map it to `shared.ErrLabAlreadyActive`.
- **`FindActiveByStudentAndCourse`**: return `(nil, nil)` when there is no active lab (active = state NOT IN done/rejected/failed). Do **not** return `ErrNotFound` here — the use case treats a `Find` miss as "no row", not an error.
- **`GetByID`**: return `(nil, shared.ErrNotFound)` when absent (this is a `Get`, miss is an error).
- **`Save`**: optimistic concurrency like `pool.go` — `UPDATE ... WHERE id = $1 AND updated_at <= $newUpdatedAt`; 0 rows affected → `shared.ErrIdempotencyClash`.
- `ki_resources` is JSONB ↔ `lab.KIResources`. `cleanup_at` / `unfreeze_at` / `frozen_by_user_id` are nullable.
- `ListPendingCleanup`: `state = 'ready' AND cleanup_at <= $before ORDER BY cleanup_at LIMIT $limit`.
- `ListPendingUnfreeze`: `state = 'frozen' AND unfreeze_at <= $before ORDER BY unfreeze_at LIMIT $limit`.

### AuditRepo

- `audit_events` has an append-only trigger — INSERT only, never UPDATE/DELETE.
- `Append` opens its own short tx (or a plain Exec); `AppendInTx` uses the caller's `tx` via `txFromPort`.
- `Query` builds a dynamic `WHERE` from `AuditFilter` (Kind, ActorUserID, Since, Limit). Order by `occurred_at DESC`. Default limit 100, cap 500.
- `audit_events.payload` is JSONB — marshal `ev.Payload` (an `any`) with `json.Marshal`.

### CourseRepo

- Straightforward. `Upsert` = `INSERT ... ON CONFLICT (external_id) DO UPDATE`. `Enroll` = `INSERT ... ON CONFLICT (user_id, course_id) DO UPDATE SET role_in_course = EXCLUDED.role_in_course`.

### QuotaCache (quota_cache.go)

- Single-row table `quota_cache (id=1, snapshot JSONB, fetched_at)`.
- `Write`: `INSERT ... ON CONFLICT (id) DO UPDATE SET snapshot = $1, fetched_at = now()`.
- `Read`: `SELECT snapshot, fetched_at FROM quota_cache WHERE id = 1`. Unmarshal snapshot JSONB → `shared.QuotaSnapshot`. `age = clock.Now() - fetched_at`. If the row is missing, return `age = 1<<62` nanoseconds (effectively infinite) so callers treat it as stale — **not** an error.
- The constructor takes a `ports.Clock` so `age` is testable: `NewQuotaCacheRepo(db *pgxpool.Pool, clk ports.Clock)`.
- Add a compile-time check `var _ lab.QuotaCache = (*QuotaCacheRepo)(nil)` — importing `internal/app/lab` for this is acceptable (the use case package owns the interface it consumes).

| Repo | Required cases |
|---|---|
