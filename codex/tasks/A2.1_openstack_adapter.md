# Task A2.1 — OpenStack CloudProvider adapter

You are a coding agent working on **Cloud Lab Gateway**, a Go backend for orchestrating cloud labs on the КИ (OpenStack-compatible) platform. Before you start, read `CLAUDE.md` and `docs/ARCHITECTURE.md` in the repo root.

## Goal

Implement `ports.CloudProvider` against the real КИ / OpenStack API using `gophercloud/v2`. The application code already calls this interface; you are filling in the production adapter.

## The interface you must satisfy

Defined in `internal/ports/cloud.go`. Methods:

```go
GetQuotaSnapshot(ctx) (shared.QuotaSnapshot, error)
CreateKeypair(ctx, projectID, name) (Keypair, error)
DeleteKeypair(ctx, projectID, name) error
BootServer(ctx, projectID, spec) (string, error)
WaitForActive(ctx, projectID, serverID) error
AllocateFloatingIP(ctx, projectID, serverID) (string, error)
DeleteServer(ctx, projectID, serverID) error
CreateNetwork(ctx, projectID, spec) (NetworkSpec) (string, error)
DeleteNetwork(ctx, projectID, networkID) error
```

## Reference implementation to mirror

`internal/adapters/cloud/inmem/provider.go` is the in-memory implementation. Your OpenStack adapter MUST exhibit the same external behaviour:

- All operations are **idempotent**. Re-creating with the same name returns the existing resource. Deleting a missing resource is not an error.
- All methods are context-cancellation-aware.
- `WaitForActive` polls and exits on either ACTIVE state or context cancellation.

## Implementation rules

### Authentication

- Auth options come from `pkg/config.OpenStack` (already populated from env).
- Use `gophercloud/v2` with `v3.AuthOptions` (Keystone v3, password auth).
- Cache the authenticated `ProviderClient` and refresh on `401`/`Unauthorized`.
- Each method requests a service client (`openstack.NewComputeV2(...)`, `openstack.NewNetworkV2(...)`) per call — gophercloud handles connection pooling.

### Quota snapshot

`GetQuotaSnapshot` MUST aggregate cluster-level utilisation, not per-project quota:

- Call Nova `os-hypervisors/statistics` for `vcpus_used`, `vcpus`, `memory_mb_used`, `memory_mb`.
- Call Cinder `os-limits` (or `os-quota-sets/<admin_project>?usage=true`) for storage.
- Return a `shared.QuotaSnapshot` with `Used`/`Total` for each dimension.

If КИ exposes a non-standard extension, prefer that; document the choice in a code comment. Falling back to per-project quota is acceptable **only** when called out explicitly.

### Idempotency

- `CreateKeypair`: look up by `name`. If exists → return existing public key. Note that OpenStack does not return the private key on lookup, so the caller must persist the private key when first created. Document this in a Go comment.
- `BootServer`: search nova servers by name within the project. If found → return the existing UUID.
- `CreateNetwork`: search neutron networks by name. If found → return existing UUID. Same for subnet + router.

### Error translation

Map gophercloud errors to domain sentinels via `internal/adapters/cloud/openstack/errors.go`:

| gophercloud type | Map to |
|---|---|
| `gophercloud.ErrDefault404` | `shared.ErrNotFound` |
| auth failure | `shared.ErrCloudUnavailable` (wrapped) |
| network timeout / 5xx | `shared.ErrCloudUnavailable` (wrapped) |
| 409 conflict | bubble up unchanged (callers handle idempotency) |

Always `fmt.Errorf("openstack: ...: %w", err)` to preserve context.

### Concurrency

The adapter is called from multiple goroutines (the saga worker + admin operations). Use `sync.RWMutex` only for the cached token; gophercloud clients themselves are safe for concurrent use.

## Tests

`provider_test.go` has two modes, gated by env vars:

1. **Smoke test against real КИ** — runs when `OPENSTACK_AUTH_URL` and `OPENSTACK_PASSWORD` are set. Boots an `m1.tiny` ubuntu cloud image, waits for ACTIVE, deletes. Skipped via `t.Skip()` when env is missing.
2. **Unit tests with a fake transport** — use `net/http/httptest.NewServer` to return canned JSON. Verify request shape (URL, body, headers, auth token included).

Do **not** require DevStack in unit tests. Anything that hits a real cluster goes behind `t.Skip()`.
