# Task B5.3 — Checks slice (CheckRunRepo, verify use cases, run-check worker, endpoints)

You are a coding agent on **Cloud Lab Gateway**. Read `CLAUDE.md` and
`docs/ARCHITECTURE.md` first.

## Goal

Wire the Ansible check engine end to end — from `POST /labs/{id}/checks` through
an async worker run to `GET /checks/{id}`. Five parts:

1. `CheckRunRepo` — Postgres persistence for `CheckRun` aggregates.
2. A minimal `CheckTemplateRepo` — resolves a check template's playbook + timeout.
3. Verify use cases — `RunCheck`, `GetCheckRun`, `ListChecksByLab`.
4. A `TaskRunCheck` worker handler — executes the check via `ports.CheckRunner`.
5. The httpapi handlers — `GET/POST /labs/{id}/checks`, `GET /checks/{id}`.

## 1 + 2 — repositories (pgxrepo)

- `CheckRunRepo` (`pgxrepo/check_run.go`): `Create` / `Save` flush nothing
  special — but persist `check_run_steps` rows alongside the `check_runs` row.
  `GetByID`, `ListByLab`. Mirror the existing repos; add `queries/check_run.sql`,
  run `make gen-sqlc`.
- `CheckTemplateRepo`: a **new minimal port** — `GetByID(ctx, id) (CheckTemplate,
  error)` returning `{PlaybookPath, TimeoutSeconds, Slug}`. Adding this interface
  to `internal/ports/storage.go` is a **coordinated ports change** — keep it
  minimal and call it out in the PR description. Implement in pgxrepo.

## 3 — verify use cases (`internal/app/verify`)

- `RunCheck(ctx, actor, labID, checkTemplateID)` — authorize (lab owner or
  teacher/admin); the lab must be `ready`; create a `CheckRun` in `queued`;
  persist; enqueue `ports.TaskRunCheck` with `{check_run_id}`. Returns the run.
- `GetCheckRun(ctx, actor, id)` / `ListChecksByLab(ctx, actor, labID)` — reads
  with the same authorization rule.

## 4 — `TaskRunCheck` worker handler

A handler (`internal/app/verify` or a sub-package) that:
- loads the `CheckRun` + its `LabInstance`;
- resolves `TargetHost` (a lab floating IP) and the checker SSH key
  (`SecretStore.Get(lab.CheckerSSHKeySecretID, "ssh_private_key", labID)`);
- resolves the playbook via `CheckTemplateRepo`;
- `CheckRun.Start(now)`, save; runs `ports.CheckRunner.Run(...)`;
  `CheckRun.Finish(result.State, summary, result.Steps, result.Stdout, now)`,
  save. Zeroize the SSH key after.
Register it in `cmd/worker/main.go` next to the deploy/cleanup handlers (A5.1).

## 5 — httpapi handlers

New file `internal/adapters/httpapi/checks.go`; remove the matching stubs from
`notimplemented.go`; extend `httpapi.Deps` with a verify-service seam.
- `POST /labs/{id}/checks` → `202` `CheckRun`.
- `GET /labs/{id}/checks` → `[]CheckRun`.
- `GET /checks/{id}` → `CheckRunDetail` (steps + `ansible_stdout`).
