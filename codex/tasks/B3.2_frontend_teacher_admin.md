# Task B3.2 — Frontend: teacher + admin consoles

You are a coding agent on **Cloud Lab Gateway**. Read `CLAUDE.md` and
`docs/ARCHITECTURE.md` first. **Your scope is `web/` only** — do not touch Go code.

## Goal

Wire the **teacher** and **admin** pages to the real API. The skeleton pages
exist (`web/src/pages/teacher-page.tsx`, `admin-page.tsx`) with a complete
layout but bound to `mock-data.ts`. Make them data-driven, following the
integration pattern established by **B3.1**.

## Depends on B3.1

B3.1 ships the typed `api` client, the query/mutation hook layer, the RFC-7807
`Problem` error helper, and the generated-types-as-source-of-truth convention.
**Reuse all of it** — same `useXxx` hook style, same toast/error handling, same
`components["schemas"]` types. Do not invent a second pattern. If B3.1 is not
merged yet, branch from it; rebase before the PR.

## Current state — study these first

- `web/src/pages/teacher-page.tsx` — renders `teacherLabs` (a labs table) and a
  "Настройки таймаутов" card from `mock-data.ts`.
- `web/src/pages/admin-page.tsx` — renders `poolGroups`, `auditEvents`,
  `quarantineProjects` and a cluster-utilisation card from `mock-data.ts`.
- `web/src/lib/mock-data.ts` — the mock arrays you are replacing.
- `api/openapi.yaml` — the contract: `Project`, `QuotaSnapshot`, `AuditEvent`,
  `Setting`, `LabInstance` schemas; the `/admin/*` and `/labs` operations.

## Teacher page

- **Course labs table** → `GET /labs?course_id=<id>` (a teacher sees the
  course's labs; optional `state` filter). Columns stay as designed: student,
  state, created, cleanup-at, last check, actions.
- **Row actions** → `POST /labs/{id}/freeze`, `POST /labs/{id}/unfreeze`,
  `POST /labs/{id}/extend` (extend takes `extend_by_seconds`; use a small
  dialog). Each mutation invalidates the labs query so the table refreshes.
- **Timeout settings** → `GET /admin/settings?scope=...` +
  `PUT /admin/settings` (the `admin-settings` tag is teacher-visible per the
  spec). A simple form; RFC-7807 validation errors shown inline.

## Admin page

- **Pool status** → `GET /admin/projects` (optional `ki_domain_id` / `state`
  filters). Group/summarise by state (free / allocated / cleaning /
  quarantine), and list the quarantined projects with their failure count and
  last error.
- **Project actions** → `POST /admin/projects/{id}/quarantine` (reason),
  `POST /admin/projects/{id}/release`.
- **Cluster utilisation** → `GET /admin/quota` (`QuotaSnapshot` —
  `utilization_pct` with a `max` used for the 90% threshold check). Reuse the
  quota-bar visual from the student page; consider extracting it to
  `components/quota-bar.tsx` if that is clean.
- **Audit log** → `GET /admin/audit` (filters: `kind`, `actor_user_id`,
  `since`, `limit`). A paginated/limited table, newest first.

## Shared rules

- Generated schema types as the source of truth; thin view-model mappers only
  for derived/formatted fields. No `any` outside `api.gen.ts`.
- Loading + error states on every async boundary; errors via the B3.1
  `Problem` helper (toast or inline).
- A `501` / `ERR_NOT_IMPLEMENTED` from a not-yet-built backend endpoint renders
  a friendly "функция скоро будет" state — never a crash. Contract-first: wire
  everything now; pages light up as the backend fills endpoints in.
- Remove the now-unused exports from `mock-data.ts` (`teacherLabs`,
  `poolGroups`, `auditEvents`, `quarantineProjects`). Keep `demoUsers` only if
  B3.1's dev-mode switch still needs it.

## Acceptance criteria

1. `npm run typecheck`, `npm run lint`, `npm run build` pass with no warnings.
2. No `any` outside `api.gen.ts`; TypeScript strict stays on.
3. Teacher and admin pages render entirely from API data — `mock-data.ts` is no
   longer imported by either page.
4. Every teacher/admin action hits the real endpoint and refreshes its view.
5. Loading + error states everywhere; errors derived from RFC-7807 `Problem`.
6. The build still serves through `deployments/Dockerfile.frontend` + nginx.

## Do NOT touch

- Anything outside `web/`.
- `api/openapi.yaml` — the contract is fixed; note any gap in the PR.
- `api.gen.ts` by hand — it is generated.
- B3.1's `lib/api.ts`, `lib/problem.ts`, hook layer — consume them, do not
  refactor them in this PR.
