# Task A4.1 — Ansible check runner + baseline playbooks

You are a coding agent on **Cloud Lab Gateway**, a Go backend. Read `CLAUDE.md` and `docs/ARCHITECTURE.md` first.

## Goal

Implement the SSH check engine — stage 3 of the case, currently at 0%. Two parts:

1. `internal/adapters/checker/ansible/` — implements `ports.CheckRunner` by executing `ansible-playbook` against a deployed lab VM and parsing the structured result.
2. `ansible/checks/` — 2-3 baseline playbooks that validate OS / software / network state.

This is fully independent of all other in-flight work — it only depends on `ports.CheckRunner`, which already exists.

## The interface (internal/ports/checker.go)

```go
type CheckRunner interface {
    Run(ctx context.Context, req CheckRequest) (CheckResult, error)
}

type CheckRequest struct {
    PlaybookPath   string
    TargetHost     string            // floating IP of the student VM
    SSHUser        string
    SSHPrivateKey  []byte            // PEM — must be zeroized after the run
    ExtraVars      map[string]string
    TimeoutSeconds int
}

type CheckResult struct {
    State  verify.RunState         // passed | failed | timeout | errored
    Steps  []verify.StepResult
    Stdout string
    Stats  AnsibleStats
}

type AnsibleStats struct{ OK, Changed, Failed, Unreachable, Skipped int }
```

`verify.RunState` and `verify.StepResult` are in `internal/domain/verify/run.go`. `StepResult` = `{TaskName string, Status StepStatus, Expected, Actual any, Message string}`; `StepStatus` ∈ ok/changed/failed/unreachable/skipped.

## Implementation rules

### runner.go

`Run` must:

1. Create a temp working dir (`os.MkdirTemp`) — remove it with `defer`.
2. Write the SSH private key to `<tmp>/key` with mode **0600**. Write an inventory file `<tmp>/inventory.ini` (mode 0600):
   ```
   [target]
   <TargetHost> ansible_user=<SSHUser> ansible_ssh_private_key_file=<tmp>/key
   ```
3. Build the command with `exec.CommandContext(ctx, "ansible-playbook", ...)`:
   - `-i <tmp>/inventory.ini`
   - the playbook path
   - `--extra-vars` from `req.ExtraVars` (JSON-encode the map)
   - SSH hardening: `--ssh-extra-args='-o StrictHostKeyChecking=accept-new -o ConnectTimeout=10'`
   - env: `ANSIBLE_STDOUT_CALLBACK=json`, `ANSIBLE_CALLBACKS_ENABLED=json`, `ANSIBLE_FORCE_COLOR=0`, `ANSIBLE_HOST_KEY_CHECKING=False`
4. Enforce `TimeoutSeconds` via a `context.WithTimeout` derived from the passed `ctx`. On timeout → `CheckResult{State: verify.StateTimeout}`, no error.
5. Capture stdout. ansible's json callback prints one big JSON document to stdout.
6. **Zeroize `req.SSHPrivateKey`** (overwrite the byte slice with zeros) before returning — use `defer`.
7. Map outcomes:
   - playbook ran, all tasks ok → `verify.StatePassed`
   - playbook ran, ≥1 task failed → `verify.StateFailed`
   - context deadline exceeded → `verify.StateTimeout`
   - ansible binary missing / unparseable output / non-playbook error → `verify.StateErrored` + return the error

### parse.go

Parse ansible's `json` stdout callback. Shape (simplified):

```json
{
  "plays": [{"tasks": [
    {"task": {"name": "..."},
     "hosts": {"<host>": {"failed": false, "changed": true, "unreachable": false, "skipped": false, "msg": "..."}}}
  ]}],
  "stats": {"<host>": {"ok": 3, "changed": 1, "failed": 0, "unreachable": 0, "skipped": 1}}
}
```

Produce one `verify.StepResult` per task (TaskName from `task.name`, Status from the host result flags, Message from `msg`), and fill `AnsibleStats` from `stats`. Keep the parser tolerant — unknown fields ignored, missing fields default sanely.

`parse_test.go` runs the parser against `testdata/callback_pass.json` and `callback_fail.json` (you author these fixtures — small, realistic). **No ansible install needed for the unit tests.**

`runner_test.go` is `//go:build integration` and exercises the real `ansible-playbook` against `localhost` (a play with `connection: local` tasks like `stat`/`package_facts`). Skip via `t.Skip()` if `ansible-playbook` is not on PATH.

### Playbooks (ansible/checks/)

Each playbook is a real, runnable check. Use `gather_facts: true` and modules that *assert* rather than *change* state:

- **linux-basics.yml** — assert: a given user exists (`getent`), a package is installed (`package_facts`), a file exists with expected content (`stat` + `slurp`), a service is running (`service_facts`). Drive expectations via `--extra-vars` (e.g. `expected_user`, `expected_package`).
- **nginx-config.yml** — assert: nginx installed + service active, listening on port 80 (`wait_for` with `timeout: 5`), a marker string present in a config file.

Use the `ansible.builtin.assert` module so a failed expectation becomes a failed task (which the runner maps to `StateFailed`). Keep them parameterised and commented — they are templates the team will clone per lab.

`ansible.cfg`: disable host key checking, set `stdout_callback = json`, `timeout = 15`.

## Constraints

- No secrets in code. The SSH key arrives via `CheckRequest`, lives only in a 0600 temp file, and is zeroized after the run.
- The temp dir must always be cleaned up, even on panic/timeout (`defer os.RemoveAll`).
- Do not shell out through `sh -c` — pass args directly to `exec.CommandContext` to avoid injection.

- Anything outside `internal/adapters/checker/` and `ansible/`.
