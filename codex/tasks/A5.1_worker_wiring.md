# Task A5.1 — Worker task wiring (deploy, cleanup, refresh-quota handlers)

You are a coding agent on **Cloud Lab Gateway**, a Go backend. Read `CLAUDE.md`
and `docs/ARCHITECTURE.md` first.

## Goal

`cmd/worker/main.go` builds an asynq server but **registers no handlers** — the
`ServeMux` is empty (see the `// Task handlers are registered here ...`
comment). The deploy and cleanup sagas are merged and tested, but nothing
dispatches to them, so no async work runs. Wire them up.

## Scope

`cmd/worker/main.go`, a small `internal/app/quotarefresh` use-case package, and
a `pkg/config` field if needed for the cloud-provider switch. Do **not** modify
the sagas or adapters.

## Handlers to register

| Task type (`internal/ports/queue.go`) | Handler |
|---|---|
| `ports.TaskDeployLab` | `deploy.Deps.HandleTask` |
| `ports.TaskCleanupLab` | `cleanup.Deps.HandleTask` |
| `ports.TaskRefreshQuota` | new — `CloudProvider.GetQuotaSnapshot` → `QuotaCache.Write` |

## Reference — mirror these (do not modify)

- `cmd/gateway/main.go` `runServe()` — the DI block (`pgxrepo.New*`,
  `clock.System`, `queueasynq.NewClient`, envkek construction). Copy the style.
- `internal/app/deploy/deps.go` + `saga.go` — `deploy.Deps` fields, `HandleTask`.
- `internal/app/cleanup/saga.go` — `cleanup.Deps` fields, `HandleTask`.
- `internal/adapters/queue/asynq` — the `Registry` / handler-binding helper that
  converts `*asynq.Task` ↔ `ports.Task`. Register `ports`-shaped handlers
  through it; do not hand-roll the conversion.
- `internal/adapters/secrets/envkek` — `KeyProvider` + `Store`.
- `internal/adapters/cloud/{inmem,openstack}` — the two `CloudProvider` impls.

## Cloud provider switch

Both sagas need a `ports.CloudProvider`. Add config `CLG_CLOUD_PROVIDER`
(`inmem` | `openstack`, default `inmem`):
- `inmem` → `inmem.New(inmem.DefaultCapacity(), inmem.Faults{})` — dev/demo path.
- `openstack` → the gophercloud adapter from its existing env config.

Log the selected provider at startup.

## Details

- Build one pgx pool, one redis client, one `clock.System`, one envkek
  `Store` — share them across both saga `Deps`.
- `deploy.Deps`: Cloud, Lab, Steps (`pgxrepo.NewDeployStepRepo`), Secrets,
  Queue, UoW, Clock, CleanupAfter. Leave `Checks` nil — the check handler is
  task B5.3.
- `cleanup.Deps`: Cloud, Lab, Pool, Secrets, UoW, Clock, Logger.
- `internal/app/quotarefresh` — a real use case (not logic in `cmd/`): a struct
  with `CloudProvider` + `QuotaCache` + a `HandleTask(ctx, ports.Task) error`
  that fetches the snapshot and writes the cache.

3. No business logic in `cmd/worker` — only wiring.
