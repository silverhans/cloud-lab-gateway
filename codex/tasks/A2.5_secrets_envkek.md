## Goal

Implement the envelope-encryption secrets subsystem — a new adapter package
`internal/adapters/secrets/envkek/` satisfying two ports:

1. `ports.KeyProvider` — the **envkek** KEK provider: the Key-Encryption-Key is
   read from the `CLG_KEK_BASE64` environment variable. It wraps/unwraps a
   per-secret DEK.
2. `ports.SecretStore` — Postgres-backed store. `Put` generates a fresh DEK,
   encrypts the payload under it (AES-256-GCM), wraps the DEK via the
   `KeyProvider`, and persists everything in `encrypted_secrets`. `Get` reverses
   it; `Delete` removes the row.

The deploy saga already calls `Secrets.Put(ctx, "ssh_private_key", labID, pem)`
(`internal/app/deploy/steps.go`, `runCreateKeypair`) and the cleanup saga calls
`Secrets.Delete` — you are providing the production implementation so SSH
private keys are stored encrypted at rest.

```go
type KeyProvider interface {
    EncryptDEK(ctx context.Context, dek []byte, aad string) (ciphertext, nonce []byte, version int, err error)
    DecryptDEK(ctx context.Context, ciphertext, nonce []byte, version int, aad string) ([]byte, error)
    CurrentVersion(ctx context.Context) (int, error)

type SecretStore interface {
    Put(ctx context.Context, kind string, refID string, payload []byte) (shared.SecretID, error)
    Get(ctx context.Context, id shared.SecretID, expectedKind, expectedRefID string) ([]byte, error)
    Delete(ctx context.Context, id shared.SecretID) error
}
```

`shared.SecretID` is a `uuid.UUID` newtype (`internal/domain/shared/ids.go`).

## Crypto design — follow exactly

Envelope encryption, two layers, both **AES-256-GCM**:

- **KEK** (Key-Encryption-Key): 32 bytes, base64-decoded from `CLG_KEK_BASE64`.
  Never persisted. Loaded once at construction; if the env var is missing or not
  exactly 32 bytes after decode → the constructor returns an error.
- **DEK** (Data-Encryption-Key): 32 bytes, generated fresh **per secret** with
  `crypto/rand`.

### KeyProvider (keyprovider.go)

- `EncryptDEK(ctx, dek, aad)`: AES-256-GCM encrypt `dek` under the KEK; `aad` is
  the GCM additional-authenticated-data; fresh 12-byte nonce from `crypto/rand`.
  Return `(ciphertext, nonce, version, nil)`.
- `DecryptDEK(ctx, ciphertext, nonce, version, aad)`: the reverse. If `version`
  != the loaded KEK version → error (single-KEK MVP, no rotation history). GCM
  open failure (tampered input / wrong aad) → error.
- `CurrentVersion`: returns `1`. The column + parameter exist so key rotation
  can be added later without a schema change.
- Constructors: `NewKeyProvider() (*KeyProvider, error)` reads `CLG_KEK_BASE64`;
  `NewKeyProviderFromKey(kek []byte) (*KeyProvider, error)` injects a known key
  so tests need no env var.

### SecretStore (store.go)

Constructor: `NewStore(db *pgxpool.Pool, kp ports.KeyProvider, audit ports.AuditRepo, clk ports.Clock) *Store`.

**AAD**: `aad = kind + ":" + refID`. It binds a ciphertext to its identity so a
row cannot be swapped for another. Compute it at `Put` time, store it in the
`aad` column, and use the **stored** value at `Get` time for both GCM opens.

- `Put(ctx, kind, refID, payload)`:
  1. Generate a 32-byte DEK with `crypto/rand`.
  2. AES-256-GCM encrypt `payload` under the DEK (aad = the computed aad) →
     `payload_ciphertext`, `payload_nonce`.
  3. `kp.EncryptDEK(ctx, dek, aad)` → `dek_ciphertext`, `dek_nonce`, `version`.
  4. **Zeroize the DEK** slice once both encryptions are done.
  5. INSERT the row; return its `shared.SecretID`.
  6. You do **not** own `payload` — do not zeroize the caller's slice; the
     caller does that.
- `Get(ctx, id, expectedKind, expectedRefID)`:
  1. SELECT the row by `id`; no row → `shared.ErrNotFound`.
  2. **Confused-deputy guard**: if `row.kind != expectedKind` OR
     `row.ref_id != expectedRefID` → `shared.ErrForbidden`. (Asking for the
     wrong secret is an authz failure, not a not-found.)
  3. `kp.DecryptDEK(...)` → DEK; then AES-256-GCM open the payload. Zeroize the
     DEK after. A GCM open failure (tampered ciphertext) → `shared.ErrSecretMismatch`.
  4. Emit a `secret.accessed` audit event via `audit.Append` — best-effort: log
     a warning on failure, do **not** fail the Get. `SubjectType = "encrypted_secret"`,
     `SubjectID = id.String()`, `ActorUserID = nil` (no principal at this layer),
     `OccurredAt = clk.Now()`. Reuse an audit-kind constant from
     `internal/domain/audit` if one exists, else the string `"secret.accessed"`.
  5. Return the plaintext; the caller zeroizes it.
- `Delete(ctx, id)`: DELETE by id. A missing row is **not** an error
  (idempotent, like the cloud-provider deletes).

## Tests

- `keyprovider_test.go` (unit — no DB, no env vars): build via
  `NewKeyProviderFromKey` with a fixed 32-byte key.
  - EncryptDEK → DecryptDEK round-trips the original DEK.
  - DecryptDEK with a flipped ciphertext byte → error.
  - DecryptDEK with the wrong `aad` → error.
  - DecryptDEK with `version = 2` → error.
  - `NewKeyProviderFromKey` rejects a key that is not 32 bytes.
- `store_test.go` (`//go:build integration`): Postgres via a `connectTestDB`
  helper (copy/adapt the one in `pgxrepo/helpers_test.go` — `PG_DSN` env, skip
  if unset; CI provides Postgres). A fake `ports.AuditRepo` and `ports.Clock`.
  - Put → Get round-trips the original payload.
  - Get with the wrong `expectedKind` → `ErrForbidden`.
  - Get on a random UUID → `ErrNotFound`.
  - Delete then Get → `ErrNotFound`; Delete twice → no error.
  - Tamper: after Put, `UPDATE` the row's `payload_ciphertext` (flip a byte),
    then Get → `ErrSecretMismatch`.
  - Fresh DEK per Put: two Puts of identical payloads produce different
    `dek_ciphertext` and `payload_ciphertext`.
