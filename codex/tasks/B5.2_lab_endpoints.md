## Use cases — add to `internal/app/lab`

Each takes the **actor** identity (user id, global role, course roles — build a
small `Actor` struct or reuse one) and enforces authorization:

| Use case | Auth | Behaviour |
|---|---|---|
| `GetLab` | owner, or teacher of the course, or admin | load `LabInstance` (+ steps for detail) |
| `ListLabs` | student → own labs; teacher → labs of a course they teach; admin → any | filter by `course_id` / `state` |
| `StopLab` | owner or teacher/admin | validate stoppable (`ready`/`failed`/`frozen`); enqueue `TaskCleanupLab`; return the lab |
| `FreezeLab` | owner ("сообщить о проблеме") or teacher/admin | `lab.Freeze(...)`; `freeze_for_seconds` (teacher only) else a default duration; save |
| `UnfreezeLab` | teacher of the course or admin | `lab.Unfreeze(cleanupAt, now)`; save |
| `ExtendLab` | teacher of the course or admin | `lab.ExtendCleanup(now+extend_by, now)`; save |
| `GetLabSSHKey` | owner only | `SecretStore.Get(lab.StudentSSHKeySecretID, "ssh_private_key", labID)` → PEM bytes |

- Authorization failure → `shared.ErrForbidden`; missing lab → `shared.ErrNotFound`.
- State-machine violations surface as `shared.ErrInvalidTransition` (→ 409).
- Saves go through `UoW.WithTx` so domain events flush to the outbox.

  and **remove the corresponding stubs from `notimplemented.go`**.
- Extend `httpapi.Deps` with a `LabOps` service seam (a narrow interface, like
  `LabService`), satisfied by the `internal/app/lab` deps.

  raw PEM body (not JSON).
- Reuse `labResponse` from `labs.go`; extend it for the detail shape.
