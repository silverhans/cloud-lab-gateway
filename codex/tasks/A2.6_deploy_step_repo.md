## Interface to satisfy (internal/ports/storage.go)

```go
type DeployStepRepo interface {
    GetOrInit(ctx context.Context, labID shared.LabInstanceID, step string) (DeployStep, error)
    Save(ctx context.Context, s DeployStep) error
    ListByLab(ctx context.Context, labID shared.LabInstanceID) ([]DeployStep, error)
}

type DeployStep struct {
    LabID      shared.LabInstanceID
    StepName   string
    Status     string
    Attempt    int
    LastError  string
    Result     []byte // JSON
    StartedAt  *time.Time
    FinishedAt *time.Time
}
```

**No `Tx` parameter — by design.** Deploy-step records are written between
multi-second cloud calls (`BootServer`, `WaitForActive`) and can never share one
database transaction. Each method is its own short statement.

## The table (already migrated — migrations/0001_init.sql)

```sql
CREATE TABLE lab_deploy_steps (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    lab_instance_id UUID NOT NULL REFERENCES lab_instances(id) ON DELETE CASCADE,
    step_name       TEXT NOT NULL CHECK (step_name IN (
        'allocate_project','create_keypair','provision_network','boot_vm',
        'wait_ssh','initial_check')),
    status          TEXT NOT NULL CHECK (status IN (
        'pending','in_progress','succeeded','failed','compensated')),
    attempt         INT NOT NULL DEFAULT 1,
    last_error      TEXT,
    result          JSONB,
    started_at      TIMESTAMPTZ,
    finished_at     TIMESTAMPTZ,
    UNIQUE (lab_instance_id, step_name)
);
```

## Implementation details

- **`GetOrInit`**: `SELECT ... WHERE lab_instance_id = $1 AND step_name = $2`.
  - Row found → map it to `DeployStep`.
  - No row → return `DeployStep{LabID: labID, StepName: step, Status: "pending",
    Attempt: 0}` with nil timestamps and nil `Result`. **Do not INSERT here** —
    `Save` does that. `Attempt: 0` matters: the saga does `Attempt++` before its
    first write, which must land on `1`.
- **`Save`**: a single UPSERT —
  ```sql
  INSERT INTO lab_deploy_steps
      (lab_instance_id, step_name, status, attempt, last_error, result, started_at, finished_at)
  VALUES (...)
  ON CONFLICT (lab_instance_id, step_name) DO UPDATE SET
      status = EXCLUDED.status, attempt = EXCLUDED.attempt,
      last_error = EXCLUDED.last_error, result = EXCLUDED.result,
      started_at = EXCLUDED.started_at, finished_at = EXCLUDED.finished_at;
  ```
- **`ListByLab`**: `SELECT ... WHERE lab_instance_id = $1 ORDER BY started_at ASC
  NULLS LAST, step_name`. Return a non-nil empty slice when there are none.
- `last_error`: nullable `TEXT` column ↔ plain `string` field. Map `"" ↔ NULL`
  (empty string stores as NULL; NULL reads back as `""`).
- `result`: nullable `JSONB` ↔ `[]byte`. Empty/nil `Result` → SQL NULL; a NULL
  row → nil `Result`.
- `started_at` / `finished_at`: nullable `TIMESTAMPTZ` ↔ `*time.Time`.
- This repo writes **no** domain events and uses **no** outbox — it is pure
  saga bookkeeping, not an aggregate. Do not mirror `pool.go`'s event flushing.

## Tests (deploy_step_test.go, //go:build integration)

Use the `connectTestDB` helper from `pgxrepo/helpers_test.go` (`PG_DSN` env,
skips if unset). A `lab_deploy_steps` row needs a parent `lab_instances` row
(FK) — insert a minimal lab first (reuse a helper if `helpers_test.go` already
provides one; otherwise add a small local one).

- `GetOrInit` on an unknown (lab, step) → fresh record, `Status` "pending",
  `Attempt` 0, nil timestamps, nil Result.
- `Save` (insert) then `GetOrInit` → returns every field intact: `Result` JSON
  round-trips, `StartedAt`/`FinishedAt` set, `LastError` preserved.
- `Save` twice for the same (lab, step) → the second is an UPDATE, not a
  duplicate row; `ListByLab` returns exactly one.
- `ListByLab` returns every step for a lab, and a non-nil empty slice for a lab
  with none.
